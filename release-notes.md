# 发布说明 · astrology v1.1.0

## 发布信息
- 版本：v1.1.0
- 发布日期：2026-07-13
- 类型：变更版本（change 模式；基于用户预览反馈迭代）
- 运行模式：纯前端静态应用，访客模式，无需注册

## 本次变更（v1.1.0）
基于用户对 v1.0.0 的预览反馈：

1. **运势改为本地预设内容**：原运势依赖外部第三方 API（aztro，个人域名），预览环境无代理导致「运势内容暂时无法获取」。现改为**本地预设内容**——12 星座 × 每日/每周各一套，含爱情/事业/健康三维度 + 星级评分。纯离线、永远可用，不再降级空态。此变更同时消除了 v1.0.0 遗留的 M1（生产代理缺失）与 M2（外部供应链风险）两项安全隐患。`aztro-provider.ts` 保留，作为日后换回真实 API 的备选。
2. **山东省城市数据补全**：山东省城市由 2 个（济南、青岛）增至 42 个，新增 14 地级市 + 26 主要县级市/市辖区。城市库总量由 75 增至 115。

## 功能概览
1. **星盘生成（本命盘）**：输入出生日期 / 时间 / 出生地，生成完整可视化星盘。涵盖十大行星落座、十二宫位、上升点及主要相位。城市识别基于离线城市库（v1.1.0 起山东省覆盖至县级市），无命中且无候选时拒绝生成并给出提示。
2. **运势板块**：展示每日 / 每周运势，含爱情/事业/健康三维度 + 星级评分。v1.1.0 起为**本地预设内容**，纯离线、永远可用。
3. **行星 / 星座百科**：22 条预置词条，支持浏览与关键词搜索；可从星盘要素直接跳转到对应百科词条（星盘—百科联动）。

## 技术栈
- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite（静态构建，产出 `dist/`）
- **样式**：Tailwind CSS，神秘暗色主题（深紫 / 深蓝背景 + 金色装饰）
- **状态管理**：Zustand（会话内存，无持久化）
- **星历计算**：纯离线 JS 星历（astronomia + luxon）
- **运势**：本地预设内容（v1.1.0 起，无外部 API 依赖）
- **形态**：纯前端静态应用，**全离线可用**，可托管于任意静态托管平台

## 部署说明
本产品为纯前端静态应用，**v1.1.0 起全离线可用、无需任何 API 代理**。部署即上传 `dist/` 目录内容：

- **Netlify**：拖拽 `dist/` 或连接 Git 仓库，Build `npm run build`、Publish `dist`。
- **Vercel**：连接 Git 仓库，Build Command `npm run build`，Output Directory `dist`。
- **GitHub Pages**：将 `dist/` 推至 `gh-pages` 分支或 `docs/`。**v1.1.0 起运势为本地内容，GitHub Pages 纯静态即可完整运行**（v1.0.0 曾因运势需代理而不适用，此限制已解除）。

> 本产品的发布落地「目标 A · GitHub 推源码」；「目标 B · 云部署」由使用者确认后手动执行，不在本阶段自动部署。
