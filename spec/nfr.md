# 非功能式样书 · astrology（SPEC-N）

> SDD 阶段 4 产出物之一。本册把 NFR-01~NFR-07 展开为可测的非功能式样。
> 编号前缀 `SPEC-N-xx`，与 NFR 一一对应。术语遵循 `discovery.md § 7`。
> 相关分册：功能见 `functional.md`（SPEC-F），画面见 `ui.md`（SPEC-U）。

---

## SPEC-N-01 性能式样
- **来源**：NFR-01
- **描述**：
  1. **端到端出盘 ≤ 60 秒**：新访客从首页加载完成到本命盘首次可见 ≤ 60 秒（含填写 3 个必填字段的用户操作时间）。
  2. **提交到渲染 ≤ 3 秒**：从点击「生成本命盘」到星盘图渲染完成 ≤ 3 秒（离线星历 + 本地渲染路径，与网络无关）。
  3. **渲染帧率**：星盘图交互（缩放/查看解读卡开合）在桌面与移动典型设备上帧率 ≥ 30 FPS，无可感知卡顿（无长任务阻塞 > 100ms 于交互后）。
  4. **计算离线**：星历计算与地理编码零外部网络请求（呼应 SPEC-F-02/F-03）。
- **验收/可测点**：
  1. 典型桌面浏览器与典型移动浏览器各计时：首页加载完成→星盘首次可见 ≤ 60 秒。
  2. 点击提交→星盘渲染完成 ≤ 3 秒（断网环境亦成立，验证离线路径）。
  3. 星盘交互操作后主线程无 > 100ms 长任务；帧率采样 ≥ 30 FPS。
- **边界与异常**：3 秒预算不含用户填表时间；运势请求（含网络）不计入出盘性能预算。

## SPEC-N-02 响应式与可用性式样
- **来源**：NFR-02
- **描述**：三板块在桌面与移动视口均可正常访问、阅读、交互。响应式断点定义：
  | 断点 | 视口宽度 | 布局 |
  |------|----------|------|
  | 移动 | `< 768px` | 单列纵向堆叠；导航可为底部标签栏或顶部横排；星盘图等比缩放至视口宽 |
  | 平板 | `768px – 1023px` | 单/双列自适应 |
  | 桌面 | `≥ 1024px` | 多列（如星盘图 + 侧栏落座清单并排） |
  - **最小可用宽度**：`320px`（在 320px 视口下无横向滚动溢出、交互元素可点）。
  - 触控目标最小可点尺寸 `≥ 44×44px`（移动可用性）。
  - 主文字与背景对比度 ≥ 4.5:1（WCAG AA，呼应 SPEC-U-01）。
- **验收/可测点**：
  1. 在 320px、375px、768px、1024px、1440px 视口下逐板块检查：无横向滚动条溢出，文字可读，交互元素可点。
  2. 星盘、运势、百科三板块在移动与桌面视口均不出现内容溢出或不可点击元素。
  3. 触控目标尺寸 ≥ 44×44px。
- **边界与异常**：`< 320px` 不作保证；星盘图在最窄视口按比例缩小但保持刻度/符号可辨。

## SPEC-N-03 术语一致性、安全与隐私式样
- **来源**：NFR-03、NFR-04
- **描述**：
  1. **术语一致性**：界面与内容所用占星术语严格遵循 `discovery.md § 7` 术语表，无同义混用（如统一用「上升星座」「宫位」「相位」「星历」「太阳星座」「本命盘」）。唯一权威来源见 SPEC-D-06。
  2. **无用户数据上传/存储**：不设账号体系；用户出生信息（日期/时间/地点）仅存活于会话内存，不做服务端持久化，不上传至任何服务器。
  3. **第三方请求不含个人身份信息**：运势 API 请求仅携带星座 + 派生日期/周，不含姓名、出生日期原文、出生地点、精确出生时间、坐标或任何可识别个人身份的数据（呼应 SPEC-A-03）。
- **验收/可测点**：
  1. 抽查三板块文案，术语与 § 7 逐项一致，无未定义/冲突术语。
  2. 提交出生信息生成星盘后，服务端无该用户出生信息的持久化记录；应用无登录态/账号数据。
  3. 抓包运势 API 请求，参数集合 ⊆ {星座, 日期/周}，不含任何个人身份信息或出生原始数据。
- **边界与异常**：出生信息不写入 localStorage/cookie/服务端存储；会话结束即不可恢复。

## SPEC-N-04 合规（免责声明）式样
- **来源**：NFR-05；FR-22
- **描述**：运势与解读内容明示「仅供娱乐参考」，避免被误作医疗/投资建议。展示位置固定为三处：每日运势视图、每周运势视图、星盘要素解读卡（与 SPEC-F-13 一致，此处从合规角度验收）。文案统一为「仅供娱乐参考」。
- **验收/可测点**：
  1. 每日运势、每周运势、星盘要素解读三处均可见「仅供娱乐参考」，且与对应内容同屏。
  2. 文案逐字为「仅供娱乐参考」，无变体（如「娱乐参考」「仅供参考」不合格）。
- **边界与异常**：运势降级空态无内容时不强制显示（无内容可免责，SPEC-F-13）。

## SPEC-N-05 可靠性与降级式样
- **来源**：NFR-07；FR-13
- **描述**：外部依赖（第三方运势 API）不可用时系统整体不崩溃：
  1. **超时阈值**：运势请求超时上限 8 秒（与 SPEC-A-03/SPEC-F-12 一致）。
  2. **降级触发**：超时 / 网络错误 / 非成功状态 / 维度缺失 / 不可解析，任一即降级。
  3. **降级行为**：运势板块显示「运势内容暂时无法获取，请稍后再试」+ 重试；不展示陈旧/错误内容。
  4. **板块隔离**：运势不可用不影响星盘生成与百科浏览（SPEC-N-07）。
- **验收/可测点**：
  1. 模拟运势 API 各失败情形，运势板块进入降级空态，应用不崩溃、不白屏。
  2. 超时精确以 8 秒为界：延迟 <8s 且成功→内容；≥8s→降级。
  3. 降级时切换到星盘/百科板块功能正常。
- **边界与异常**：系统不自动无限重试；重试为用户手动触发。

## SPEC-N-06 可维护性式样
- **来源**：NFR-06
- **描述**：内容与代码逻辑解耦：
  1. **百科**为预置静态内容数据（首版无后台，假设 A-2）；词条数据独立于星历计算与渲染代码。
  2. **运势**来自第三方 API，来源切换不影响星盘与百科板块。
  3. 内容更新（改一条百科词条 / 切换运势来源）不需改动核心计算或渲染逻辑代码。
- **验收/可测点**：
  1. 修改一条百科静态词条数据后，词条详情随之更新，且无需改动星历/渲染代码文件。
  2. 运势来源切换（替换 API 实现/地址）不影响星盘生成与百科浏览。
  3. 百科内容以独立数据文件（如 JSON/数据模块）维护，与逻辑代码分离（结构见 SPEC-D-05）。
- **边界与异常**：宫位/相位解释作为解读卡内联文案（假设 A-1），不属可独立维护的词条数据。

## SPEC-N-07 板块隔离与容错式样
- **来源**：NFR-07
- **描述**：三板块（星盘/运势/百科）功能相互隔离，单一外部依赖故障不扩散：星盘与百科为纯本地（离线可用），不依赖运势 API；运势板块故障被局部捕获，仅在运势板块降级，不使整页崩溃或阻断导航。
- **验收/可测点**：
  1. 断网状态下：星盘可生成、百科可浏览（纯本地路径），运势进入降级空态。
  2. 运势请求抛错被局部捕获，全局导航与其余两板块仍可用。
  3. 任一板块异常不导致整页白屏。
- **边界与异常**：地理编码与星历为离线本地，故不构成外部故障点（呼应约束 C-2/C-3）。
