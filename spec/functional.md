### C-PLANET 十大行星（固定顺序与符号）
太阳 ☉、月亮 ☽、水星 ☿、金星 ♀、火星 ♂、木星 ♃、土星 ♄、天王星 ♅、海王星 ♆、冥王星 ♇。

### C-SIGN 黄道十二星座（固定顺序，起点白羊 0°）
白羊、金牛、双子、巨蟹、狮子、处女、天秤、天蝎、射手、摩羯、水瓶、双鱼。每星座占黄经 30°。

### C-ASPECT 主要相位与容许角度（orb）
| 相位 | 精确角度 | 容许 orb |
|------|----------|----------|
| 合相 | 0° | ±8° |
| 六分相 | 60° | ±6° |
| 四分相 | 90° | ±8° |
| 三分相 | 120° | ±8° |
| 对分相 | 180° | ±8° |

> orb 阈值关闭要件 UQ-2 遗留项。两行星的黄经差落入 `精确角度 ± orb` 区间即判定为该相位；若同时落入多个区间，取角度差最小者。

### C-HOUSE 宫位制
采用 **Placidus（普拉西德）宫位制**；第一宫宫头 = 上升点。降级盘（无出生时间）不计算宫位。

## SPEC-F-01 出生信息表单校验规则
- **来源**：FR-01、FR-05；natal-chart.feature「缺少出生日期时无法生成本命盘」
- **描述**：出生信息表单采集且仅采集三类输入：出生日期、出生时间、出生地点。校验规则如下：
  1. **出生日期**：必填。格式为公历年-月-日，年份范围 `1900-01-01` 至 `当前系统日期`（含）。缺失、非法格式、超出范围三种情形均视为校验不通过。
  2. **出生时间**：可选。为 24 小时制「时:分」，分钟粒度。提供独立的「时间未知」开关；开关开启时，时间输入被禁用并视为未提供（触发 SPEC-F-04 降级）。
  3. **出生地点**：必填。为文本输入，值必须能被城市库解析（见 SPEC-F-02 / SPEC-A-01）；解析失败视为校验不通过。
- **验收/可测点**：
  1. 表单恰好包含 3 个输入项（日期、时间、地点）与 1 个「时间未知」开关，无其它出生相关字段。
  2. 出生日期为空时提交被拒，显示提示「出生日期为必填项」，且不触发任何星历计算。
  3. 出生日期 `< 1900-01-01` 或 `> 当前系统日期` 时提交被拒，显示提示「出生日期超出可用范围」。
  4. 「时间未知」开关开启时，时间输入不可编辑，且提交成功（进入 SPEC-F-04 降级）。
  5. 出生地点为空时提交被拒，显示提示「请选择出生地点」。
- **边界与异常**：
  - 出生日期 = 当前系统日期：允许。
  - 出生日期 = `1900-01-01`：允许；`1899-12-31`：拒绝。
  - 出生时间 `24:00` 或 `23:60`：非法格式，拒绝提交（允许 `00:00`~`23:59`）。
  - 闰年 `2月29日`：合法；非闰年 `2月29日`：非法格式，拒绝。

## SPEC-F-02 地点离线地理编码流程
- **来源**：FR-02、FR-04；natal-chart.feature 背景、「出生地点无法识别时不生成本命盘」；约束 C-3
- **描述**：系统将出生地点文本解析为经纬度与时区，全程使用内置城市库（见 SPEC-D-03），不发起任何外部网络请求。解析流程：
  1. 对输入文本做规范化（去首尾空白；全角转半角）。
  2. 在城市库中做精确名称匹配，命中即返回该条目的经度、纬度、时区标识（IANA 时区名，如 `Asia/Shanghai`）。
  3. 未精确命中时，按城市库提供的别名/前缀候选返回候选列表供用户选择（见 SPEC-U-04）；用户未选定候选前不进入星历计算。
  4. 既无精确命中也无候选时，判定为「无法识别」，走 SPEC-F-06 拒绝路径。
- **验收/可测点**：
  1. 输入城市库中存在的城市名，返回其经度、纬度、时区，且解析过程零外部网络请求（可通过网络监控断言无出站请求）。
  2. 同一城市名连续解析两次，返回的经度、纬度、时区完全一致（支撑 FR-23 / SPEC-F-08）。
  3. 输入城市库中不存在且无候选的地点，返回「无法识别」结果码，不返回任何坐标。
- **边界与异常**：
  - 空字符串：不进入本流程，由 SPEC-F-01 拦截。
  - 同名城市（多条同名条目）：返回候选列表，由用户选定唯一条目后再计算。
  - 时区解析结果用于将「出生本地时间」转换为 UTC 供星历计算（见 SPEC-F-03）。

## SPEC-F-03 星历计算流程（含降级路径）
- **来源**：FR-06、FR-03；natal-chart.feature「提供完整出生信息后生成完整本命盘」「出生时间未知时给出降级的本命盘」；约束 C-2、C-5
- **描述**：给定合法出生日期、（可选）出生时间、经纬度、时区，系统在本地完成星历计算，产出星盘结果（SPEC-D-02）。流程与两条路径：
  1. **时刻构造**：将出生本地日期时间按解析出的时区转换为 UTC 时刻，再转为儒略日（用于星历库输入）。
  2. **完整路径（出生时间已知）**：
     - 计算 C-PLANET 十大行星的黄经，映射为「所在星座 + 星座内度数」（C-SIGN）。
     - 计算上升点（ASC）与中天（MC）的黄经与落座星座。
     - 按 C-HOUSE（Placidus）计算十二宫宫头，据此定位每颗行星的所在宫位。
     - 按 C-ASPECT 计算行星两两之间的主要相位列表。
     - `hasTime = true`，`ascendantAvailable = true`，`housesAvailable = true`。
  3. **降级路径（出生时间未知）**：
     - 仅计算十大行星的黄经与落座星座（对当日 `12:00` 的地点本地时间做一次计算以确定星座落座）。
     - 上升、中天、十二宫、相位均不计算。
     - `hasTime = false`，`ascendantAvailable = false`，`housesAvailable = false`；上升与宫位字段值为「不可用」标识（非 null 数值，见 SPEC-D-02）。
  4. 全部计算在本地完成，不依赖外部星历 API。
- **验收/可测点**：
  1. 完整路径下，输出覆盖十大行星（各含星座 + 度数 + 宫位）、上升点、中天点、相位列表。
  2. 降级路径下，输出十大行星的落座星座；上升、中天、宫位标注为「不可用」；相位列表为空；不因缺时间报错。
  3. 完整路径中每颗行星的宫位取值在 1~12 之间且唯一映射（同一行星只属于一个宫）。
  4. 相位判定符合 C-ASPECT：黄经差落入区间即入列，多区间时取最小角差。
  5. 计算全程零外部网络请求。
- **边界与异常**：
  - 降级路径以「当日正午」定位太阳星座，因太阳日移动约 1°，边界日太阳星座以此结果为准并与 C-SUN 表可能相差一日——降级盘显示星历结果，不显示 C-SUN 表值。
  - 行星黄经恰为 `0°`（白羊起点）：归入白羊 `0°00'`。
  - 两行星黄经差恰为 `orb` 边界值（如 8°00'）：判定为该相位（区间闭合）。

## SPEC-F-04 出生时间未知时的降级星盘
- **来源**：FR-03、FR-08；约束 C-5；natal-chart.feature「出生时间未知时给出降级的本命盘」
- **描述**：当出生日期与可识别地点齐备、「时间未知」开关开启时，系统生成降级本命盘。降级盘的构成：
  1. 展示太阳星座与十大行星各自的落座星座（来自 SPEC-F-03 降级路径）。
  2. 上升星座、中天、十二宫在星盘图与行星落座清单中以「不可用」呈现（见 SPEC-U-05），不显示任何推测数值。
  3. 行星落座清单每行显示「行星 → 星座」，「宫位」列显示「不可用」。
- **验收/可测点**：
  1. 齐备日期 + 可识别地点 + 时间未知，提交后成功输出降级盘，无阻断、无报错。
  2. 输出含十大行星的落座星座与太阳星座。
  3. 上升与宫位区域明确显示「不可用」文案，星盘图不绘制上升点标记与宫位分隔线。
- **边界与异常**：降级盘同样满足稳定可复现（SPEC-F-08）：同一「日期 + 地点 + 时间未知」两次生成，十大行星落座星座一致。

## SPEC-F-05 太阳星座判定规则
- **来源**：FR-07；natal-chart.feature 场景大纲「依据出生日期判定太阳星座」
- **描述**：太阳星座判定分两个场景：
  1. **表单即时/降级判定**：仅依据出生日期（月-日），按 C-SUN 边界日期表归属。
  2. **完整盘判定**：以 SPEC-F-03 计算出的太阳黄经落座星座为准。
- **验收/可测点**：
  1. 场景大纲逐条通过（仅依日期，C-SUN 表）：3月21日→白羊、6月22日→巨蟹、8月15日→狮子、11月10日→天蝎、12月25日→摩羯。
  2. C-SUN 表对 1月1日至12月31日的任一日期均返回唯一星座，无空档、无重叠。
- **边界与异常**：
  - 摩羯跨年（12月22日–次年1月19日）：`12月31日`→摩羯，`1月1日`→摩羯，`1月19日`→摩羯，`1月20日`→水瓶。
  - 各边界起始日（如 3月21日→白羊、3月20日→双鱼）严格按 C-SUN 表闭区间归属。
  - 完整盘中若星历太阳落座与 C-SUN 表在边界日不一致，星盘最终显示星历结果；C-SUN 表值仅用于无时间/无地点时的即时提示。

## SPEC-F-06 地点无法识别时拒绝生成
- **来源**：FR-04；natal-chart.feature「出生地点无法识别时不生成本命盘」
- **描述**：当出生地点经 SPEC-F-02 判定为「无法识别」时，系统不进入星历计算、不生成任何本命盘输出，并提示用户重选城市。
- **验收/可测点**：
  1. 提交含无法识别地点的出生信息后，无任何星盘结果对象产生，星盘视图保持在表单/提示态。
  2. 显示提示文案「出生地点无法识别，请重新选择城市」。
- **边界与异常**：日期与时间即便完全合法，只要地点无法识别即拒绝生成（地点校验优先级高于生成）。

## SPEC-F-07 本命盘渲染要求
- **来源**：FR-08；natal-chart.feature「提供完整出生信息后生成完整本命盘」
- **描述**：系统将星盘结果（SPEC-D-02）渲染为一张圆形可视化本命盘，并同步给出行星落座清单。完整盘必须呈现的元素：
  1. **黄道十二星座刻度**：外圈按 C-SIGN 顺序等分 12 段，每段标注星座符号。
  2. **十大行星**：按各自黄经在盘上定位，标注 C-PLANET 行星符号。
  3. **十二宫**：以宫头分隔线划分十二个扇区，标注宫位序号 1~12。
  4. **上升点（ASC）**：在盘的东方（左侧）水平位置标注上升点标记与其落座星座。
  5. **主要相位连线**：在盘内圈以连线表示 SPEC-F-03 计算出的相位（连线样式区分相位类型，见 SPEC-U-05）。
  6. **行星落座清单**：星盘图下方/侧方列出十行，每行含「行星名、所在星座、所在宫位」。
- **验收/可测点**：
  1. 完整盘渲染后，页面同时可见：12 星座刻度、10 行星符号、12 宫分隔、1 个上升点标记、≥0 条相位连线、10 行落座清单。
  2. 落座清单每行三列非空（行星名、星座、宫位 1~12）。
  3. 降级盘渲染时：不绘制上升点标记与宫位分隔线，落座清单「宫位」列显示「不可用」，星座刻度与行星符号仍绘制。
- **边界与异常**：无相位时相位连线为 0 条，星盘图其余元素照常渲染。

## SPEC-F-08 稳定可复现规则
- **来源**：FR-23；natal-chart.feature「相同出生信息产生稳定可复现的本命盘」
- **描述**：星历计算与地理编码必须是纯确定性的：给定完全相同的出生信息（日期 + 时间 + 已选定的同一城市条目），多次生成的星盘结果在十大行星落座（星座 + 宫位）与上升星座上完全一致，计算过程不得引入随机数、当前时钟、或未固定的浮点非确定性。
- **验收/可测点**：
  1. 同一份完整出生信息两次生成，两次结果的十大行星星座、宫位、上升星座逐项相等。
  2. 降级盘两次生成，十大行星落座星座逐项相等。
  3. 计算不读取 `当前系统时间`（当前时间仅用于运势子域，不进入星历计算）。
- **边界与异常**：同名城市须先由用户选定唯一条目，才能保证坐标一致进而结果一致（见 SPEC-F-02）。

## SPEC-F-09 星盘要素解读（与百科联动）
- **来源**：FR-09、FR-19；natal-chart.feature「查看星盘要素的解读」；encyclopedia.feature「从星盘要素跳转到对应百科词条」
- **描述**：在已生成的本命盘上，用户查看某行星落座含义时，系统展示与百科词条一致的解读文案，并附免责声明：
  1. 查看某行星「落座星座」的含义时，展示该星座百科词条（SPEC-D-05）的解释文本，与 SPEC-F-11 百科详情所示同源、逐字一致。
  2. 解读卡内包含「仅供娱乐参考」免责声明（见 SPEC-F-13 / SPEC-N-04）。
  3. 联动不离开访客模式、不需要登录、不做页面级跳转即可查看（就地展示解读卡）。
- **验收/可测点**：
  1. 从本命盘某行星落座请求查看其所在星座含义，展示的解释文本与该星座百科词条 `explanation` 字段一致。
  2. 解读卡内可见「仅供娱乐参考」字样。
  3. 整个联动过程无登录/注册提示，用户身份仍为访客。
- **边界与异常**：降级盘同样支持查看行星落座星座解读（星座已知）；宫位「不可用」时不提供宫位解读入口。

## SPEC-F-10 运势按日期自动滚动规则
- **来源**：FR-10、FR-11、FR-14；horoscope.feature 背景、各运势场景、「跨日后运势自动滚动到当日」、场景大纲
- **描述**：运势子域依据「当前系统日期」自动对应当期内容：
  1. **每日运势**：目标日期 = 当前系统日期（用户本地日历日）。每日运势内容维度至少含爱情、事业、健康三项（其余维度以第三方 API 实际返回为准，见 SPEC-A-03、假设 A-4）。
  2. **每周运势**：目标周 = 当前系统日期所在的自然周（周一至周日为一周，周一为起始日）。
  3. **自动滚动**：每次进入或刷新运势视图时，均以「当次查看时刻的系统日期」重新计算目标日期/目标周，不缓存展示昨日/上周内容。
  4. **星座严格对应**：请求星座与返回并展示的运势星座必须相同。
- **验收/可测点**：
  1. 选择某星座每日运势，展示的运势日期标签等于当前系统日期。
  2. 系统日期滚动到新一天后再次查看同一星座每日运势，展示的是当日运势，日期标签更新，非昨日内容。
  3. 切换到每周运势，展示周与当前系统日期所在的周一至周日区间一致。
  4. 场景大纲逐条通过：查看白羊/金牛/双子/天秤/双鱼每日运势，返回运势星座与所选星座一致。
- **边界与异常**：
  - 跨周边界（周日 23:59 → 周一 00:00）：周一起以新周为目标周。
  - 每日运势维度不足三项（API 未返回）：视为内容不完整，按 SPEC-F-12 降级处理（不展示残缺内容）。

## SPEC-F-11 百科浏览与检索规则
- **来源**：FR-15、FR-16、FR-17、FR-18；encyclopedia.feature 各场景与两场景大纲
- **描述**：百科覆盖十二星座（C-SIGN）与十大行星（C-PLANET）共 22 个词条（SPEC-D-05）。功能规则：
  1. **词条详情**：浏览任一星座或行星词条，展示其符号、象征、关键词、解释四类内容。
  2. **关键词搜索**：输入关键词，返回名称或关键词字段包含该关键词（不区分大小写、子串匹配）的词条列表，每条结果标识其类别（星座/行星）。
  3. **空结果**：无匹配时返回空列表并提示「未找到相关词条」。
- **验收/可测点**：
  1. 十二星座、十大行星词条齐全（共 22 条），任一词条的符号、象征、关键词、解释四项均非空。
  2. 搜索存在匹配的关键词，返回列表含该词条并标识类别。
  3. 场景大纲「按名称查到对应类别」逐条通过：太阳→行星、月亮→行星、水星→行星、天蝎→星座、摩羯→星座。
  4. 搜索无匹配关键词，返回空列表并显示「未找到相关词条」。
- **边界与异常**：
  - 空搜索词：返回全部 22 条词条（等价于浏览全部），不显示空结果提示。
  - 关键词首尾空白：搜索前去除首尾空白后匹配。

## SPEC-F-12 第三方运势 API 请求与降级策略
- **来源**：FR-12、FR-13；NFR-07；horoscope.feature 背景、「运势内容来源不可用时降级提示」；约束 C-4
- **描述**：每日/每周运势内容取自第三方运势 API（SPEC-A-03），系统「打开即见」当期内容，不主动推送任何通知（无邮件/短信/push）。请求与降级策略：
  1. **请求触发**：仅在用户打开或刷新对应运势视图时发起请求，入参为「星座 + 目标日期/目标周」。
  2. **超时阈值**：单次请求超时上限 **8 秒**（见 SPEC-N-05）；超时按失败处理。
  3. **降级触发条件**（任一成立即降级）：请求超时、网络错误、非成功响应状态、响应体缺失必需维度（爱情/事业/健康未全）、响应无法解析。
  4. **降级行为**：不展示任何过期或错误的运势内容，改为显示提示「运势内容暂时无法获取，请稍后再试」，并提供重试入口。
  5. **隔离性**：运势 API 不可用不影响星盘与百科板块（见 SPEC-N-07）。
- **验收/可测点**：
  1. 正常响应时，展示的运势文案来自 API 响应，而非本地硬编码占位内容。
  2. 系统运行期间不产生任何邮件/短信/push 通知出站。
  3. 模拟 API 超时（>8 秒）、错误状态、维度缺失三种情形，均不展示运势内容，显示「运势内容暂时无法获取，请稍后再试」并可重试。
  4. 运势 API 不可用时，切换到星盘/百科板块仍可正常生成星盘、浏览词条。
- **边界与异常**：
  - 请求携带的参数不含任何用户个人身份信息（仅星座 + 日期，见 SPEC-N-03/隐私）。
  - 重试为用户手动触发；系统不自动无限重试。

## SPEC-F-13 免责声明呈现规则
- **来源**：FR-22；NFR-05；natal-chart.feature「查看星盘要素的解读」；horoscope.feature 每日/每周运势场景；约束 C-6
- **描述**：系统在三处内容区域呈现「仅供娱乐参考」免责声明，且声明与对应内容同屏可见、无需额外操作：
  1. 每日运势视图（运势卡内或紧邻位置）。
  2. 每周运势视图（运势卡内或紧邻位置）。
  3. 星盘要素解读卡（SPEC-F-09）。
- **验收/可测点**：
  1. 每日运势、每周运势、星盘要素解读三处均显示「仅供娱乐参考」字样。
  2. 免责声明与其对应内容处于同一可视区域，展示内容时即可见，无需滚动/点击额外元素。
- **边界与异常**：运势降级态（SPEC-F-12）不展示运势内容，此时不强制显示免责声明（无内容可免责）。

## SPEC-F-14 访客模式规则
- **来源**：FR-20；NFR-04；三份 feature 背景「访客模式」；约束 C-1
- **描述**：用户无需注册或登录即可使用星盘、运势、百科全部核心功能。系统不提供账号体系，不做服务端持久化保存用户出生信息与历史星盘。
- **验收/可测点**：
  1. 首次进入即可访问三大板块，全程无注册/登录步骤与入口。
  2. 界面无登录/注册/账号相关功能项。
  3. 提交出生信息生成星盘后，服务端不存在该用户出生信息或历史星盘的持久化记录（另见 SPEC-N-03 隐私式样）。
- **边界与异常**：出生信息仅存在于当前浏览器会话内存中用于生成星盘；会话结束即不可恢复（不做本地持久存储或云端保存）。

## SPEC-F-15 全局导航与默认板块规则
- **来源**：FR-21；navigation.feature 全部四场景
- **描述**：系统提供全局导航，含且仅含「星盘 / 运势 / 百科」三项，支持任意板块间直接切换：
  1. **默认板块**：首次打开应用、加载完成后默认呈现「星盘」板块，当前板块标识为「星盘」。
  2. **切换**：从任一板块可通过导航直达其余两个板块，切换后当前板块标识更新为目标板块名。
  3. **状态保持**：在同一会话内切换板块再切回时，各板块已有的用户输入/结果状态保持（如已生成的星盘、已选运势星座、已浏览的词条），不因切换而丢失或重置；切换不重新加载整页应用。
- **验收/可测点**：
  1. 应用加载完成后当前板块为「星盘」，星盘板块内容可见。
  2. 从星盘切到运势，当前板块标识变为「运势」，运势内容可见；再切到百科同理；三板块两两可达。
  3. 生成星盘后切到运势再切回星盘，先前生成的星盘仍在（未被清空重置）。
  4. 切换板块不触发整页刷新（单页应用内切换）。
- **边界与异常**：导航项恰为三项，无第四板块入口；无账号/设置等额外导航项（呼应 SPEC-F-14）。
